# Demo UI Blocks Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> executing-plans to implement this plan task-by-task. Each task follows TDD
> (test-driven-development) and uses ide-tooling for structural editing.
> Steps use checkbox (`- [ ]`) syntax for tracking.

**Focal issue:** #60 — chore: migrate demo UI to blocks-ui components
**Issue group:** #60, #66, #67, #68, #69

**Goal:** Replace three hand-rolled Lit demo UI components with blocks-ui shared components and migrate all CSS tokens from `--blocks-*` to `--pages-*`.

**Architecture:** Sequential migration — CSS tokens first (§69), then layout (§66), channel feed (§67), approval gate (§68). Each step is independently testable. The approval gate migration includes a backend endpoint change (new `PUT .../workitems/{gateId}/complete`, remove old `POST .../approve|reject`).

**Tech Stack:** Lit 3, `@casehubio/blocks-ui-*` packages, `@casehubio/pages-ui-tokens`, `@casehubio/pages-primitives`, Quarkus (Java 21), esbuild

## Global Constraints

- **Pre-release platform** — breaking changes cost nothing. Remove dead endpoints, don't keep them for backwards compatibility.
- **IntelliJ MCP mandatory** for all `.java` and `.ts` edits — use `ide_edit_member`, `ide_replace_member`, `ide_insert_member`, `ide_create_file`, `ide_refactor_safe_delete`. Never use Edit/Write on existing source files.
- **Custom element collision** — `customElements.define()` throws on duplicate names. When replacing a local component with a blocks-ui equivalent that uses the same tag, remove the local import *before* adding the blocks-ui import. Never import both simultaneously.
- **pages event bus** — blocks-ui components emit events via `emitPagesEvent()` (topic-based `CustomEvent('pages-event')`), not standard DOM events. Subscribe with `onPagesEvent()` from `@casehubio/blocks-ui-core`.
- **Token verification** — before applying the token mapping, verify exact property names against the current `@casehubio/pages-ui-tokens` source. The mapping in the spec was validated at design time but tokens may have shifted.
- **npm registry** — `.npmrc` already configures `@casehubio:registry=https://npm.pkg.github.com`. No changes needed.
